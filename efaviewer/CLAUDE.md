# EFA Viewer

A web-based viewer for rowing club logbook data exported from EFA (Elektronisches Fahrtenbuch).

## Architecture

- **`app/`** - Static single-page web app (vanilla HTML/CSS/JS, no build step)
  - `index.html` - Main page with tabs: On Water, Logbook, Statistics
  - `app.js` - All application logic (data loading, filtering, charts, tables)
  - `styles.css` - All styles, uses CSS custom properties for theming
  - `data/` - JSON data files generated by the importer (not tracked in git)
  - `node_modules/@maxpoletto/sortable-table` - Custom table component (npm package)
- **`bin/`** - Python CLI tools
  - `efa_importer.py` - Converts EFA XML backup files → JSON for the web app
  - `efa_viewer.py` - CLI tool for viewing/analyzing EFA XML files directly
- **`import.sh`** - Fetches latest EFA backup from server and runs the importer

## Data flow

1. `import.sh` downloads EFA backup (zip of XML files) from `efa.belvoir-rc.ch`
2. `bin/efa_importer.py` parses XML, resolves IDs, exports `{boats,persons,destinations,logbooks}.json.gz`
3. [For development] `import.sh` decompresses the `.gz` files into `app/data/`
4. The web app fetches the JSON files at load time

## Key conventions

- Data is stored as arrays of arrays (not objects) in the JS app for compactness. Column indices are defined as constants (`YEAR_COLUMN`, `DATE_COLUMN`, etc.) at the top of `app.js`.
- Entity IDs from EFA are UUIDs. Boats use a variant suffix: `{uuid}-v{n}`. "Former" entities (referenced by name in old logs, not in the current EFA database) get synthetic IDs like `former-000001`.
- Dates in EFA data are DD.MM.YYYY (European format).
- The app uses nouislider for range sliders and Chart.js for bar charts. These are loaded from CDN.
- The `@maxpoletto/sortable-table` component is a custom npm package loaded directly from `node_modules/` (no bundler).

## Running locally

```bash
# Import fresh data (requires SSH access to efa.belvoir-rc.ch)
./import.sh

# Serve the app (any static server works)
cd app && python3 -m http.server 8000
```

## Dependencies

- **Frontend (CDN):** nouislider 15.7.x, Chart.js 4.4.x, chartjs-adapter-date-fns 3.x
- **Frontend (npm):** @maxpoletto/sortable-table
- **Backend:** Python 3 (standard library only)
